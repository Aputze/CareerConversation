SECTION 2: AI INTEGRATION ARCHITECTURE (4-LAYER MODEL)
---------------------------------------------------------------------

Sergei designed and built a production-ready 4-layer integration architecture connecting Priority ERP to modern LLMs:

Layer 1 — User Intent:
• Natural language input from the user (free text).
• Intent classification and routing.

Layer 2 — Knowledge Layer (ERP Ontology):
• What does Priority know about this domain?
• Entity capabilities, lifecycle rules, routing intelligence.
• What can and cannot be done via which interface path?
• Sources: Priority official documentation, community knowledge base (RAG), CLAUDE.md governance rules.

Layer 3 — Heuristic Layer (Pre-flight):
• Pre-flight decision engine: which tool, which endpoint, which pattern?
• OData vs WebSDK routing logic.
• Is this a known failure path? Does a tested pattern exist?
• Governance rules enforcement before any execution.

Layer 4 — Operational Layer (MCP Gateway):
• Execution with full business-logic awareness.
• Not "write field X" but "run the process that updates X."
• OData for CRUD operations.
• WebSDK for procedures, form lifecycle, reports.
• Human approval checkpoint on every sensitive recommendation.
• Full traceability: write-back linked to the originating Priority entity.
• Payload logging (in and out) for stability monitoring and periodic regression testing.
